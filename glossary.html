📖 The AI Dictionary 🐱

Every AI term explained simply with cat-themed analogies and real-world use cases. Click any term's lesson link to dive deeper.

A B C D E F G H I K L M N O P Q R S T U V W Z

A

Accuracy

A metric measuring how often a model's predictions are correct, calculated as the number of correct predictions divided by the total number of predictions.

🐱 If Whiskers correctly identifies 9 out of 10 treats as "tuna" vs "salmon," her accuracy is 90%.

💼 Use case: Measuring how well an image classifier correctly labels photos of cats vs dogs.

Agent (AI Agent)

An AI system that autonomously perceives its environment, makes decisions, and takes actions to achieve a goal — often using tools like web search or code execution.

🐱 Agent Whiskers independently hunts for information, checks the fridge, and orders tuna — all without being told each step.

💼 Use case: An AI assistant that browses the web, reads emails, and books meetings on your behalf.

→ Lesson 17: AI Agents

Artificial Intelligence

The broad field of computer science focused on creating systems that can perform tasks that normally require human intelligence, such as reasoning, learning, and problem-solving.

🐱 AI is like teaching a robot cat to do everything a real cat does — nap strategically, judge humans, and occasionally be useful.

💼 Use case: Powering virtual assistants, recommendation engines, self-driving cars, and medical diagnostics.

→ Lesson 01: What is AI?

Attention Mechanism

A technique that allows a model to focus on the most relevant parts of the input when producing each part of the output, rather than treating all input equally.

🐱 When Whiskers hears "treat," she tunes out everything else and zeros in on that one word — that's attention.

💼 Use case: Helping a translation model focus on the right source words when generating each target word.

→ Lesson 09: Transformer Architecture

Autoencoder

A neural network that learns to compress data into a compact representation (encoding) and then reconstruct it (decoding), used for dimensionality reduction and anomaly detection.

🐱 Whiskers summarizes her day into five meows, then her kitten reconstructs the full story from those meows.

💼 Use case: Detecting fraudulent transactions by learning what "normal" looks like and flagging deviations.

B

Backpropagation

The algorithm used to train neural networks by computing gradients of the loss function with respect to each weight, then propagating them backwards through the network to update weights.

🐱 Whiskers misses a jump. She retraces every decision — angle, speed, timing — to figure out exactly where she went wrong.

💼 Use case: Teaching a neural network to classify handwritten digits by correcting its mistakes layer by layer.

→ Lesson 04: Neural Networks

Batch Size

The number of training examples processed together in one forward and backward pass before updating the model's weights.

🐱 Instead of correcting Whiskers after every single toy toss, you let her play with 32 toys then give one big coaching session.

💼 Use case: Choosing a batch size of 64 to balance training speed and memory usage when training an image classifier.

BERT

Bidirectional Encoder Representations from Transformers — a pre-trained language model by Google that reads text in both directions simultaneously to understand context deeply.

🐱 Whiskers reads a sentence from both left and right at the same time, understanding the middle word using clues from all sides.

💼 Use case: Powering Google Search to understand the intent behind user queries, not just keywords.

→ Lesson 09: Transformer Architecture

Bias (Model Bias)

Systematic errors in a model's predictions caused by unfair assumptions in the training data or algorithm, leading to unfair or skewed outcomes for certain groups.

🐱 If Whiskers only trained on tabby cats, she'll wrongly label all Siamese cats as "not cat" — that's bias from skewed data.

💼 Use case: Hiring algorithms unfairly penalizing candidates from underrepresented groups due to biased historical data.

→ Lesson 18: Model Bias & Fairness

Bounding Box

A rectangle drawn around an object in an image to indicate its location, used in object detection tasks alongside a class label.

🐱 Whiskers draws a box around every bird she spots through the window — that box is a bounding box.

💼 Use case: Self-driving cars drawing bounding boxes around pedestrians, traffic lights, and other vehicles in real time.

C

Classification

A supervised learning task where the model assigns an input to one of a predefined set of categories or classes.

🐱 Whiskers looks at every item in the house and sorts it into: "toy," "food," or "ignore" — that's classification.

💼 Use case: Spam filters classifying incoming emails as "spam" or "not spam."

CNN (Convolutional Neural Network)

A neural network architecture designed for processing grid-structured data like images, using convolutional layers to detect local patterns such as edges, textures, and shapes.

🐱 A cat's eye scans a scene in overlapping patches — spotting whiskers here, ears there — just like a CNN slides a filter across an image.

💼 Use case: Face recognition on your phone, medical image analysis, and autonomous driving cameras.

→ Lesson 08: CNNs — Computer Vision

Context Window

The maximum number of tokens an LLM can process in a single interaction — both the input prompt and the generated output combined must fit within this limit.

🐱 Whiskers has a short memory — she only remembers the last 10 minutes of conversation. That's her context window.

💼 Use case: A model with a 128K token context window can process an entire novel at once before generating a summary.

→ Lesson 10: Large Language Models

Corpus

A large, structured collection of text used to train or evaluate language models, ranging from web pages and books to domain-specific documents.

🐱 Whiskers's library of every cat meow ever recorded — thousands of purrs, hisses, and chirps — is her training corpus.

💼 Use case: The Common Crawl web corpus (petabytes of text) used to pre-train GPT-style models.

Cross-Entropy Loss

A loss function that measures how well a model's predicted probability distribution matches the true distribution, commonly used for classification tasks.

🐱 Whiskers confidently says "90% tuna, 10% salmon" but it's actually salmon — cross-entropy penalizes her confidence in the wrong answer.

💼 Use case: Training image classifiers and language models to output calibrated probability scores.

D

Dataset

A structured collection of examples (inputs and sometimes labels) used to train, validate, or test a machine learning model.

🐱 Prof. Whiskers's stack of 10,000 flashcards with cat photos on one side and breed names on the other — that's a dataset.

💼 Use case: ImageNet, a dataset of 14 million labeled images, used to train early deep learning vision models.

→ Lesson 03: Training Data & Datasets

Deep Learning

A subset of machine learning using neural networks with many layers (deep architectures) to automatically learn hierarchical representations from raw data.

🐱 A kitten first learns edges, then shapes, then faces — each layer of understanding builds on the last, going deeper and deeper.

💼 Use case: Powering voice assistants, image generation, medical diagnosis, and language translation.

→ Lesson 05: Deep Learning

Diffusion Model

A generative model that learns to create data by learning to reverse a process of gradually adding noise, starting from pure noise and iteratively denoising to produce a clean output.

🐱 A static-filled TV screen slowly resolves into a crisp picture of Whiskers, step by step — that's diffusion in reverse.

💼 Use case: Stable Diffusion and DALL-E 3 generating photorealistic images from text prompts.

→ Lesson 12: Diffusion Models

Discriminator (GAN)

The component of a Generative Adversarial Network that evaluates whether a given sample is real (from the training data) or fake (produced by the Generator).

🐱 The Cat Critic looks at each painting and judges: "Real masterpiece or Whiskers's forgery?" — pushing the artist to improve.

💼 Use case: In GAN-based image synthesis, the discriminator forces the generator to produce ever-more-realistic faces.

→ Lesson 11: GANs

Dropout

A regularization technique where random neurons are temporarily deactivated during training to prevent overfitting and force the network to learn more robust features.

🐱 Each training session, some kittens in the class randomly fall asleep — the remaining ones must still solve the problem without them.

💼 Use case: Applied in deep networks to reduce reliance on specific neurons and improve generalization to new data.

→ Lesson 05: Deep Learning

E

Embedding

A dense numerical vector representation of data (words, sentences, images) in a continuous space where semantically similar items are placed close together.

🐱 "Kitten" and "cat" sit close together on Whiskers's mental map, while "kitten" and "spaceship" are far apart — that's an embedding space.

💼 Use case: Powering semantic search so that "feline" and "cat" return the same results, and enabling recommendation systems.

→ Lesson 14: Tokenization & Embeddings

Encoder

The component of a model that transforms raw input (text, images, audio) into a rich internal representation (usually a vector or sequence of vectors) that captures meaning.

🐱 Whiskers reads the entire scene outside the window and compresses it into a single "danger level" score in her head — that's encoding.

💼 Use case: BERT uses an encoder to convert sentences into embeddings for downstream tasks like classification.

→ Lesson 09: Transformer Architecture

Epoch

One complete pass through the entire training dataset during model training. Models are typically trained for multiple epochs until performance converges.

🐱 Whiskers reviews all 1,000 flashcards once — that's one epoch. She does it 50 times to really memorize them.

💼 Use case: Training a neural network for 100 epochs while monitoring validation loss to decide when to stop.

→ Lesson 05: Deep Learning

Evaluation Metrics

Quantitative measures used to assess model performance, such as accuracy, precision, recall, F1-score, BLEU, or ROUGE, chosen based on the task and what matters most.

🐱 Whiskers's report card — not just "did she catch the toy?" but also "how fast?" and "how often did she miss?" — covers multiple metrics.

💼 Use case: Using F1-score instead of accuracy when class imbalance exists, like in rare disease detection.

F

Feature

An individual measurable property or characteristic of the data used as input to a model, such as pixel values, word frequencies, or sensor readings.

🐱 When identifying a cat, Whiskers notes: pointy ears, whiskers, fur texture, tail length — each of these is a feature.

💼 Use case: A house price model uses features like square footage, location, and number of bedrooms.

Feature Extraction

The process of using a pre-trained model to transform raw input into meaningful intermediate representations (features) that can be used for a new task without full retraining.

🐱 Whiskers uses her cat-hunting skills (detecting movement, shapes) as a base to spot birds — she's extracting and reusing features.

💼 Use case: Using a pre-trained ResNet's convolutional layers to extract image features for a custom medical imaging classifier.

→ Lesson 07: Transfer Learning & Fine-tuning

Fine-tuning

The process of taking a pre-trained model and continuing training on a smaller, task-specific dataset to adapt its knowledge to a new domain or task.

🐱 Whiskers already knows how to hunt mice; with a few lessons, she adapts those skills to herd sheep. Same base skills, new specialty.

💼 Use case: Fine-tuning GPT on company documents to create a customer service chatbot that knows your product.

→ Lesson 07: Transfer Learning & Fine-tuning

Foundation Model

A large model trained on broad data at massive scale that can be adapted to a wide range of downstream tasks, serving as a foundation for many applications.

🐱 Whiskers is so universally talented she can hunt, herd, cuddle, or perform — foundation models are the Whiskers of AI.

💼 Use case: GPT-4, Claude, and Gemini are foundation models used for writing, coding, analysis, and more.

→ Lesson 10: Large Language Models

Function Calling

A capability allowing LLMs to invoke external tools or APIs in a structured way — the model decides when and how to call a function based on the conversation context.

🐱 Agent Whiskers knows she can ring the treat bell, call for backup, or check the weather app — function calling is knowing which button to press.

💼 Use case: A travel assistant LLM calling a flight search API to return real-time prices within a chat conversation.

→ Lesson 17: AI Agents

G

GAN (Generative Adversarial Network)

A framework where two neural networks — a Generator and a Discriminator — compete against each other: the generator tries to create realistic fake data, while the discriminator tries to detect fakes.

🐱 Cat Artist creates paintings; Cat Critic tries to spot forgeries. Their competition forces the artist to get so good the critic can't tell.

💼 Use case: Generating photorealistic human faces (StyleGAN), deepfakes, and data augmentation for rare medical cases.

→ Lesson 11: GANs

Generative AI

AI systems that can create new content — text, images, audio, video, or code — that didn't exist before, by learning patterns from training data.

🐱 Instead of just recognizing cat photos, Whiskers can now paint entirely new cat portraits she's never seen before — that's generative.

💼 Use case: ChatGPT writing essays, Midjourney generating artwork, and GitHub Copilot completing code.

GPU

Graphics Processing Unit — hardware originally designed for rendering graphics that, due to its massively parallel architecture, is ideal for the matrix computations in deep learning.

🐱 A CPU is one very smart cat solving puzzles one by one; a GPU is 10,000 average cats solving the same puzzle simultaneously.

💼 Use case: Training GPT-3 required thousands of NVIDIA A100 GPUs running in parallel for months.

Gradient Descent

An optimization algorithm that iteratively adjusts model weights in the direction that most reduces the loss function, like descending a hill to find the lowest point.

🐱 Whiskers is lost in hilly terrain and always takes a step downhill — eventually she reaches the valley (minimum error).

💼 Use case: The core algorithm behind training nearly every modern neural network, from tiny classifiers to massive LLMs.

Grounding

The process of anchoring an AI model's outputs to factual, verifiable sources — preventing hallucination by tying responses to real, retrieved information.

🐱 Instead of guessing what's in the fridge, Whiskers opens it and checks before answering — grounded in reality, not imagination.

💼 Use case: RAG systems ground LLM answers in retrieved documents so the model doesn't fabricate facts.

→ Lesson 16: RAG Systems

H

Hallucination (AI)

When an AI model generates confident-sounding but factually incorrect or completely fabricated information, presenting it as if it were true.

🐱 Whiskers tells everyone she climbed Mount Everest last Tuesday — convincingly and with detail — but she's never left the apartment.

💼 Use case: A chatbot confidently citing a scientific paper that doesn't exist — a dangerous hallucination in medical or legal contexts.

→ Lesson 19: AI Hallucinations

Hyperparameter

Configuration settings set before training begins (not learned from data) that control the training process, such as learning rate, batch size, number of layers, and dropout rate.

🐱 Before Whiskers starts studying, you decide: how many hours per day, how many flashcards per session, how long to rest — those are hyperparameters.

💼 Use case: Tuning the learning rate via grid search to find the best value that makes training converge fastest.

Hugging Face

An open-source AI platform and community that hosts thousands of pre-trained models, datasets, and tools, making state-of-the-art AI accessible to developers worldwide.

🐱 Hugging Face is the giant library where Whiskers can borrow any pre-trained cat-brain off the shelf and take it home immediately.

💼 Use case: Downloading a pre-trained BERT model in two lines of Python code to classify sentiment in customer reviews.

I

Inference

The process of using a trained model to generate predictions or outputs on new, unseen data — as opposed to training, where the model learns from data.

🐱 Whiskers is done studying. Now when she sees a new animal, she just identifies it on the spot — that's inference, no more studying needed.

💼 Use case: Running a trained fraud detection model in real time on incoming bank transactions to flag suspicious ones.

Instruction Tuning

Fine-tuning a language model on examples of instruction-following tasks (e.g., "Summarize this," "Translate that") to make it better at following natural language instructions.

🐱 Whiskers was a great natural learner, but instruction tuning teaches her to follow commands like "sit," "fetch," and "explain quantum physics simply."

💼 Use case: InstructGPT and ChatGPT were trained with instruction tuning to be helpful assistants rather than raw text predictors.

K

Knowledge Cutoff

The date after which an LLM has no training data — events, discoveries, or changes that occurred after this date are unknown to the model unless retrieved externally.

🐱 Whiskers went into a cave in April 2024. She knows everything that happened before then, but nothing after — that's her knowledge cutoff.

💼 Use case: A model with a 2023 cutoff won't know about 2024 elections or product releases unless you tell it or use RAG.

L

Label

The correct answer or category associated with a training example in supervised learning — what the model is trained to predict.

🐱 Each flashcard has a cat photo on one side and the breed name written on the back — that breed name is the label.

💼 Use case: Human annotators labeling thousands of medical images as "tumor" or "no tumor" to train a cancer detection model.

→ Lesson 03: Training Data & Datasets

Latent Space

A compressed, abstract representation of data learned by a model — a lower-dimensional space where similar concepts are clustered near each other.

🐱 Whiskers's mental map of all animals she knows — cats cluster together, dogs in another area, and mythical creatures off in a weird corner.

💼 Use case: In image generation, navigating latent space lets you smoothly morph a cat face into a dog face.

→ Lesson 12: Diffusion Models

Layer (Neural Network)

A group of neurons in a neural network that processes its inputs together — networks stack multiple layers (input, hidden, output) to learn increasingly abstract features.

🐱 First layer detects Whiskers's edges, second detects her features, third recognizes her face — each layer adds understanding.

💼 Use case: A deep network for speech recognition with dozens of layers, each learning progressively more abstract audio patterns.

→ Lesson 04: Neural Networks

Learning Rate

A hyperparameter that controls how large the steps are when updating model weights during gradient descent — too high causes instability, too low causes slow convergence.

🐱 A huge learning rate is Whiskers bounding down the hill and overshooting the valley; too small and she barely moves — the sweet spot finds the bottom.

💼 Use case: Learning rate schedulers that start high and decay over training time are standard practice in modern deep learning.

LLM (Large Language Model)

A neural network with billions of parameters trained on massive text corpora to understand and generate human language, capable of tasks from writing to coding to reasoning.

🐱 The Literary Cat who read every book ever written — she can continue any story, answer any question, and write in any style.

💼 Use case: ChatGPT, Claude, and Gemini powering chatbots, code assistants, document summarizers, and more.

→ Lesson 10: Large Language Models

LoRA (Low-Rank Adaptation)

An efficient fine-tuning technique that adds small, trainable rank-decomposition matrices to a frozen pre-trained model, drastically reducing the number of parameters needed to fine-tune.

🐱 Instead of retraining all of Whiskers's skills from scratch, you add tiny sticky notes to key skills — lightweight but effective updates.

💼 Use case: Fine-tuning a 70B parameter LLM on a single GPU by only training LoRA adapter weights (a few million parameters).

→ Lesson 07: Transfer Learning & Fine-tuning

Loss Function

A mathematical function that measures how wrong the model's predictions are — training aims to minimize this value so the model improves its accuracy.

🐱 Whiskers's disappointment score after guessing wrong — the higher the score, the worse she did, and the more she needs to adjust.

💼 Use case: Mean Squared Error for regression tasks, Cross-Entropy for classification — choosing the right loss matters enormously.

M

Machine Learning

A subset of AI where systems learn patterns from data and improve their performance over time without being explicitly programmed for each task.

🐱 Instead of writing rules for Whiskers, you show her 1,000 cat photos and let her figure out what "cat" means on her own.

💼 Use case: Email spam filters, Netflix recommendations, and credit scoring all use machine learning.

→ Lesson 02: Machine Learning Basics

Multi-Head Attention

An extension of the attention mechanism that runs multiple attention operations in parallel, each learning to focus on different aspects of the input simultaneously.

🐱 Whiskers reads a sentence with eight pairs of eyes at once — one pair focuses on grammar, another on meaning, another on tone — all simultaneously.

💼 Use case: Transformers use multi-head attention so the model simultaneously attends to syntax, semantics, and co-reference in text.

→ Lesson 09: Transformer Architecture

Multimodal

AI models that can process and generate multiple types of data — such as text, images, audio, and video — within a single unified architecture.

🐱 Whiskers can read a caption, look at a photo, listen to a meow, and respond in all three formats — she's multimodal.

💼 Use case: GPT-4V, Gemini, and Claude analyze images alongside text, enabling use cases like chart reading and visual Q&A.

N

Neural Network

A computational model loosely inspired by biological brains, consisting of interconnected layers of nodes (neurons) that transform inputs into outputs through learned weights.

🐱 Whiskers's brain is a network of neurons passing notes to each other — each neuron says "I spotted something suspicious" and votes on the final answer.

💼 Use case: The backbone of image recognition, speech synthesis, language translation, and game-playing AI.

→ Lesson 04: Neural Networks

NLP (Natural Language Processing)

The field of AI focused on enabling computers to understand, interpret, and generate human language — encompassing tasks like translation, summarization, and sentiment analysis.

🐱 Teaching a computer to understand Whiskers's meows, hisses, and purrs — every nuance of feline (or human) language.

💼 Use case: Powering Google Translate, autocomplete, customer service chatbots, and contract analysis tools.

Node (Neuron)

The basic computational unit of a neural network that receives inputs, applies an activation function, and passes its output to the next layer.

🐱 Each neuron-kitten in class receives notes from classmates, decides how excited to get (activation), and passes a note to the next row.

💼 Use case: A deep network may have millions of nodes, each contributing to recognizing a specific pattern or feature.

→ Lesson 04: Neural Networks

O

Overfitting

When a model learns the training data too well — including its noise and quirks — so it performs great on training data but poorly on new, unseen data.

🐱 Whiskers memorizes every exact question from last year's exam perfectly, then fails the new exam because she only memorized, not understood.

💼 Use case: A model that achieves 99% training accuracy but only 60% test accuracy — a classic overfitting sign.

→ Lesson 06: Overfitting vs. Underfitting

Optimization

The process of adjusting a model's parameters to minimize the loss function, using algorithms like gradient descent, Adam, or RMSProp.

🐱 Whiskers keeps tweaking her hunting strategy — adjusting pounce angle, timing, and approach — until she catches the toy every time.

💼 Use case: The Adam optimizer is the default choice for training most modern deep learning models due to its adaptive learning rates.

P

Parameters

The learnable values (weights and biases) inside a neural network that are adjusted during training. The number of parameters reflects a model's capacity to learn complex patterns.

🐱 Every instinct Whiskers has trained — how much to weight each clue — are her parameters. GPT-4 has hundreds of billions of them.

💼 Use case: GPT-3 has 175 billion parameters; more parameters generally means more capability but also more compute and memory.

Perplexity

A metric for language models measuring how surprised the model is by new text — lower perplexity means the model is better at predicting the text, indicating higher quality.

🐱 Whiskers reads a sentence and guesses the next word. If she's rarely surprised, her perplexity is low — she's a great predictor.

💼 Use case: Comparing language models: a model with perplexity 20 is better than one with perplexity 50 on the same test set.

Pooling

A downsampling operation in CNNs that reduces the spatial dimensions of feature maps by summarizing regions (e.g., max pooling takes the maximum value in each region).

🐱 Whiskers scans a 4×4 patch of the photo and just notes the most distinctive detail — pooling keeps the gist and discards the noise.

💼 Use case: Max pooling reduces a 224×224 image to smaller representations while retaining dominant features like edges and shapes.

→ Lesson 08: CNNs — Computer Vision

Pretraining

The initial phase of training a large model on massive unlabeled datasets using self-supervised objectives (like next-token prediction), building general knowledge before task-specific fine-tuning.

🐱 Whiskers spends years reading every book in the library — no specific goal, just absorbing everything — before specializing in one topic.

💼 Use case: GPT-4 was pre-trained on trillions of tokens before being fine-tuned for chat and instruction-following.

Prompt

The input text provided to a language model to guide its output — the instructions, context, examples, or questions that shape what the model generates.

🐱 Whiskers won't move until you tell her exactly what you want. A vague "do something" gets chaos; a clear prompt gets results.

💼 Use case: Crafting a detailed prompt for a code assistant to generate production-ready Python with error handling and tests.

→ Lesson 15: Prompt Engineering

Prompt Engineering

The practice of designing, iterating, and optimizing input prompts to elicit better, more accurate, or more useful outputs from AI language models.

🐱 Whispering "Whiskers, please bring me the red toy near the couch" vs. just "fetch" — the specificity of your words determines the quality of the result.

💼 Use case: Using chain-of-thought prompting ("let's think step by step") to dramatically improve LLM reasoning on math problems.

→ Lesson 15: Prompt Engineering

Q

Quantization

A technique to reduce model size and inference cost by representing weights and activations with lower-precision numbers (e.g., 4-bit integers instead of 32-bit floats) with minimal accuracy loss.

🐱 Instead of Whiskers remembering every detail in full HD, she stores memories in compressed JPEG — smaller, nearly as good, fits on less hardware.

💼 Use case: Running a 70B parameter LLM on a laptop using 4-bit quantization (GGUF format) instead of requiring a server.

Query (Attention)

In the attention mechanism, the Query is a vector representing "what am I looking for?" — it's compared against Keys to determine how much attention to pay to each Value in the input.

🐱 Whiskers thinks "I'm looking for danger" (her Query), scans the room for things matching that (Keys), and weighs the relevant ones (Values).

💼 Use case: Every transformer layer computes queries, keys, and values to route information through the network when processing language.

→ Lesson 09: Transformer Architecture

R

RAG (Retrieval-Augmented Generation)

A technique that combines an LLM with a retrieval system — the model first searches an external knowledge base for relevant documents, then uses them as context to generate a grounded response.

🐱 Whiskers checks the library before answering — she retrieves the relevant books first, then crafts her answer based on what she actually read.

💼 Use case: Enterprise chatbots that answer questions about internal documents without hallucinating or going out-of-date.

→ Lesson 16: RAG Systems

Reinforcement Learning

A type of machine learning where an agent learns to make decisions by taking actions in an environment and receiving rewards or penalties, optimizing for long-term cumulative reward.

🐱 Cat in a maze gets a treat for going right, a spray of water for going left — she learns the optimal path purely through trial and reward.

💼 Use case: Training AlphaGo to master the game of Go, and teaching robots to walk without falling over.

→ Lesson 13: Reinforcement Learning

RLHF (Reinforcement Learning from Human Feedback)

A training technique where human raters compare model outputs, a reward model is trained on their preferences, and the LLM is fine-tuned using RL to produce outputs humans prefer.

🐱 Whiskers shows two meow performances; judges vote on the better one. She learns to perform the crowd-pleasing version — that's RLHF.

💼 Use case: ChatGPT and Claude were aligned using RLHF to be helpful, harmless, and honest rather than just statistically plausible.

→ Lesson 13: Reinforcement Learning

RNN (Recurrent Neural Network)

A neural network architecture designed for sequential data that maintains a hidden state across time steps, allowing it to process inputs one at a time while remembering past context.

🐱 Whiskers reads a sentence one word at a time, updating her mental state after each word — she never forgets earlier words as she processes later ones.

💼 Use case: Largely replaced by Transformers, but still used for time-series forecasting and some sequential data tasks.

ReLU (Rectified Linear Unit)

The most common activation function in deep learning — it outputs the input directly if positive, and zero otherwise, introducing non-linearity without vanishing gradient problems.

🐱 Whiskers ignores signals below zero (no reaction) but reacts proportionally to positive signals — simple, fast, and very effective.

💼 Use case: Used as the activation function in most hidden layers of CNNs and deep feedforward networks.

S

Self-Attention

An attention mechanism where each element in a sequence attends to all other elements in the same sequence, allowing the model to capture relationships and dependencies within the input.

🐱 Reading "the cat sat on the mat because it was soft," Whiskers automatically connects "it" to "mat" — that cross-referencing is self-attention.

💼 Use case: The core mechanism in Transformer models, enabling understanding of long-range dependencies in text.

→ Lesson 09: Transformer Architecture

Semantic Search

A search approach that finds results based on meaning and intent rather than exact keyword matching, using embeddings to measure conceptual similarity between query and documents.

🐱 Whiskers searches for "feline" and finds "cat" results — she's searching by meaning, not just the exact word.

💼 Use case: Enterprise search tools that find relevant documents even when the exact search terms don't appear in them.

→ Lesson 14: Tokenization & Embeddings

Softmax

An activation function that converts a vector of raw scores into a probability distribution — all values sum to 1 — commonly used in the output layer of classification models.

🐱 Whiskers rates "tuna" 8, "salmon" 5, "kibble" 2 — softmax converts these to 73%, 24%, 3% probabilities. She's 73% sure it's tuna.

💼 Use case: The final layer of an image classifier assigns probabilities across 1,000 ImageNet classes using softmax.

Supervised Learning

A machine learning paradigm where the model is trained on labeled examples — each input is paired with the correct output — and learns to map inputs to outputs.

🐱 Prof. Whiskers shows flashcards with photos AND answers on the back — the student learns by seeing both the question and correct answer.

💼 Use case: Spam detection, image classification, sentiment analysis, and medical diagnosis models all use supervised learning.

→ Lesson 02: Machine Learning Basics

System Prompt

A special instruction given to an LLM before any user interaction that sets its persona, behavior, constraints, and context — invisible to users but shaping all responses.

🐱 Before Whiskers meets a visitor, you whisper: "Act professional, don't scratch, speak only about cats." That whisper is the system prompt.

💼 Use case: Customer service chatbots use system prompts to restrict the model to company-relevant topics and maintain brand voice.

→ Lesson 15: Prompt Engineering

T

Temperature (LLM)

A parameter that controls the randomness of an LLM's output — low temperature (near 0) makes outputs more focused and deterministic; high temperature makes them more creative and varied.

🐱 Temperature 0.0 is sleepy Whiskers giving safe, predictable answers. Temperature 1.5 is her after too much catnip — creative, unpredictable, and occasionally unhinged.

💼 Use case: Use low temperature for factual Q&A, high temperature for brainstorming and creative writing.

Token

The basic unit of text that language models process — typically a word, sub-word, or character. "Tokenization" is roughly 1 token per 0.75 words in English.

🐱 Whiskers doesn't read whole words; she reads syllable-chunks. "Unbelievable" becomes "Un-believ-able" — three tokens, three bites of text.

💼 Use case: LLM pricing is typically per 1,000 tokens — understanding tokenization helps manage API costs.

→ Lesson 14: Tokenization & Embeddings

Tokenization

The process of splitting raw text into tokens (sub-words, words, or characters) that a model can process numerically, using algorithms like BPE (Byte Pair Encoding) or WordPiece.

🐱 Whiskers chops a long sentence into bite-sized pieces before eating it — tokenization is the chopping, tokens are the bites.

💼 Use case: GPT models use BPE tokenization with a vocabulary of ~50,000 tokens to handle any text input.

→ Lesson 14: Tokenization & Embeddings

Training Data

The labeled (or unlabeled) examples used to teach a model — the quality, quantity, and diversity of training data is the single biggest factor in model performance.

🐱 Prof. Whiskers's flashcards are her training data. Garbage flashcards (blurry photos, wrong labels) produce a confused student.

💼 Use case: GPT-4 was trained on trillions of tokens; curated, high-quality data matters more than sheer volume.

→ Lesson 03: Training Data & Datasets

Transfer Learning

A technique where a model trained on one task is reused as the starting point for a different but related task, dramatically reducing data and compute requirements.

🐱 Whiskers mastered hunting mice; now she transfers those stalking, pouncing, and timing skills to herd sheep with minimal retraining.

💼 Use case: Fine-tuning a pre-trained BERT model for medical text classification with only a few thousand labeled examples.

→ Lesson 07: Transfer Learning & Fine-tuning

Transformer

A neural network architecture based entirely on attention mechanisms (introduced in "Attention Is All You Need," 2017) that processes sequences in parallel and forms the backbone of all modern LLMs.

🐱 Unlike RNN-Whiskers reading word-by-word, Transformer-Whiskers reads the whole sentence at once and pays attention to every word's relationship with every other word.

💼 Use case: The architecture behind GPT, BERT, T5, LLaMA — virtually every state-of-the-art language, vision, and multimodal model.

→ Lesson 09: Transformer Architecture

U

Underfitting

When a model is too simple to capture the underlying patterns in the data, performing poorly on both training and test sets — the model hasn't learned enough.

🐱 Whiskers only learned "if it has fur, it's a cat" — she fails on dogs, lions, and teddy bears. She's underfitting from a too-simple rule.

💼 Use case: A linear model trying to fit complex non-linear data will underfit — visible as poor performance even on training data.

→ Lesson 06: Overfitting vs. Underfitting

Unsupervised Learning

A machine learning paradigm where the model finds patterns in unlabeled data without any correct answers provided — discovering structure, clusters, or representations on its own.

🐱 No flashcard answers this time — Whiskers just groups all the animals herself based on what she notices. She discovers "fluffy ones" and "scaly ones" on her own.

💼 Use case: Customer segmentation, anomaly detection, and dimensionality reduction (PCA) for data exploration.

V

Validation Set

A subset of data held out during training, used to tune hyperparameters and monitor model performance — distinct from the test set (used only for final evaluation).

🐱 Whiskers practices with flashcards (training), checks her progress with pop quizzes (validation), and takes the final exam once (test).

💼 Use case: Monitoring validation loss during training to detect overfitting and trigger early stopping.

Vector

A mathematical object represented as an ordered list of numbers — in AI, vectors encode information like word meanings, image features, or document content as coordinates in high-dimensional space.

🐱 Whiskers's personality in coordinates: [curiosity: 9.2, laziness: 7.8, friendliness: 4.1] — that's a vector representation of a cat.

💼 Use case: Text embeddings are vectors; "king" - "man" + "woman" ≈ "queen" — arithmetic works in vector space.

Vector Database

A specialized database optimized for storing, indexing, and efficiently searching high-dimensional embedding vectors using approximate nearest neighbor algorithms.

🐱 Whiskers's library where every book is stored as coordinates — ask for "something near adventure + mystery" and get the closest matches instantly.

💼 Use case: Pinecone, Weaviate, and pgvector power the retrieval layer of RAG systems, finding relevant documents from millions of embeddings in milliseconds.

→ Lesson 16: RAG Systems

Vision Transformer (ViT)

A Transformer architecture applied to images by splitting them into fixed-size patches and treating each patch as a token, achieving state-of-the-art results on image classification.

🐱 Whiskers cuts her photo into a grid of 16×16 tiles, reads each tile as a "word," and understands the full picture using attention — same trick as text Transformers.

💼 Use case: ViT models power Google's image search and form the vision component of multimodal models like GPT-4V.

W

Weight (Neural Network)

A learnable parameter in a neural network that determines the strength of the connection between neurons — weights are adjusted during training to minimize the loss function.

🐱 Each neuron-kitten has a volume knob for how loudly to pass a signal. Training slowly turns the right knobs up and the wrong ones down.

💼 Use case: A neural network's weights are what gets saved when you download a model file — they encode everything the model learned.

→ Lesson 04: Neural Networks

Z

Zero-Shot Learning

The ability of a model to correctly perform a task it has never been explicitly trained on, relying solely on its general knowledge and the task description provided in the prompt.

🐱 Nobody taught Whiskers about capybaras, but she correctly identifies one just from the description "large, friendly, semi-aquatic rodent" — that's zero-shot.

💼 Use case: Asking GPT-4 to classify a new product category it's never seen by describing the categories in the prompt alone.
