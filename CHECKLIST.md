# ✅ Referencia Content Checklist

Use this checklist when creating each new lesson to ensure quality and consistency.

## Pre-Production

### Planning Phase
- [ ] Define lesson topic and scope
- [ ] Identify target audience level (Beginner/Intermediate/Advanced)
- [ ] List 3-5 key learning objectives
- [ ] Estimate completion time (30-120 minutes)
- [ ] Outline main sections and subsections
- [ ] Identify prerequisite knowledge
- [ ] Plan 2-3 hands-on experiments

### Research Phase
- [ ] Gather authoritative sources
- [ ] Find quality video tutorials
- [ ] Identify useful tools/platforms
- [ ] Select relevant research papers
- [ ] Bookmark community resources
- [ ] Review existing lessons for consistency

## Content Creation

### Lesson Structure
- [ ] Write compelling lesson title
- [ ] Create engaging subtitle/description
- [ ] Set lesson metadata (time, level, sections)
- [ ] Write overview section with objectives
- [ ] Draft core content sections
- [ ] Create accordion items for complex topics
- [ ] Add code examples (if applicable)
- [ ] Write clear explanations for each concept

### Visual Content
- [ ] Plan sketchnote layout
- [ ] Identify key visual metaphors
- [ ] Create sketchnote draft
- [ ] Refine and finalize sketchnote
- [ ] Export at correct dimensions (1200x900px+)
- [ ] Save to `/assets/sketchnotes/`
- [ ] Update HTML with image path
- [ ] Add descriptive alt text

### Experiments Section
- [ ] Identify 2-3 relevant experiments
- [ ] Build or adapt demos
- [ ] Deploy to hosting platform
- [ ] Test experiment functionality
- [ ] Write experiment descriptions
- [ ] List concepts covered
- [ ] Add working links
- [ ] Verify links open correctly

### Resources Section
- [ ] Curate 4-6 quality articles
- [ ] Select 3-4 video tutorials
- [ ] List 3-4 useful tools
- [ ] Recommend 2-3 books
- [ ] Add community links (optional)
- [ ] Verify all links are active
- [ ] Organize by category

## Technical Implementation

### HTML Structure
- [ ] Copy lesson template
- [ ] Update page title and meta description
- [ ] Set lesson number
- [ ] Add lesson title with emoji
- [ ] Update lesson info (time, level, sections)
- [ ] Populate overview section
- [ ] Add sketchnote container
- [ ] Fill core content
- [ ] Add experiment cards
- [ ] Populate resources
- [ ] Update navigation links (prev/next)

### Content Formatting
- [ ] Use proper heading hierarchy (h2, h3, h4)
- [ ] Apply bold/strong to key terms
- [ ] Format code blocks (if any)
- [ ] Add lists where appropriate
- [ ] Include relevant emojis
- [ ] Add internal section links
- [ ] Ensure proper spacing

### Interactive Elements
- [ ] Add accordion items for detailed content
- [ ] Test accordion expand/collapse
- [ ] Verify smooth scrolling
- [ ] Check section navigation
- [ ] Test experiment links
- [ ] Verify resource links open in new tabs

## Quality Assurance

### Content Review
- [ ] Proofread entire lesson
- [ ] Check grammar and spelling
- [ ] Verify technical accuracy
- [ ] Ensure logical flow
- [ ] Check for consistent terminology
- [ ] Validate code examples work
- [ ] Confirm examples are clear

### Technical Testing
- [ ] Test on Chrome
- [ ] Test on Firefox
- [ ] Test on Safari
- [ ] Test on mobile (iOS)
- [ ] Test on mobile (Android)
- [ ] Verify responsive layout
- [ ] Check all links work
- [ ] Test navigation buttons
- [ ] Verify images load
- [ ] Check accessibility (keyboard nav)

### Cross-Device Check
- [ ] Desktop (1920x1080)
- [ ] Laptop (1366x768)
- [ ] Tablet (iPad, 768x1024)
- [ ] Mobile (375x667)
- [ ] Mobile landscape

## Integration

### Add to Site
- [ ] Update main `index.html` lesson grid
- [ ] Add lesson card with correct info
- [ ] Update lesson number sequence
- [ ] Fix navigation on adjacent lessons
- [ ] Update sitemap (if using)
- [ ] Add to search index (if applicable)

### SEO & Meta
- [ ] Add descriptive page title
- [ ] Write compelling meta description
- [ ] Add relevant keywords
- [ ] Create social media preview image
- [ ] Add Open Graph tags (optional)
- [ ] Add Twitter Card tags (optional)

## Pre-Launch

### Final Checks
- [ ] Re-read entire lesson
- [ ] Test all interactive elements
- [ ] Verify all resources accessible
- [ ] Check mobile experience
- [ ] Test load time
- [ ] Validate HTML (https://validator.w3.org)
- [ ] Check for broken links
- [ ] Review with fresh eyes after 24 hours

### Peer Review (Recommended)
- [ ] Share with 1-2 colleagues
- [ ] Gather feedback
- [ ] Make revisions based on feedback
- [ ] Test clarity with target audience
- [ ] Verify learning objectives achieved

## Post-Launch

### Monitoring
- [ ] Check analytics (page views, time on page)
- [ ] Monitor for broken links
- [ ] Track user feedback
- [ ] Note common questions
- [ ] Identify areas for improvement

### Maintenance
- [ ] Update outdated information quarterly
- [ ] Refresh external links semi-annually
- [ ] Add new experiments as available
- [ ] Incorporate user suggestions
- [ ] Keep resources current

## Optional Enhancements

### Advanced Features
- [ ] Add video embeds
- [ ] Create interactive diagrams
- [ ] Add quiz questions
- [ ] Include downloadable resources
- [ ] Create practice exercises
- [ ] Add code sandboxes
- [ ] Include discussion prompts

### Community
- [ ] Enable comments (if desired)
- [ ] Add sharing buttons
- [ ] Create discussion topics
- [ ] Encourage contributions
- [ ] Feature community experiments

## Lesson-Specific Notes

### For Beginner Lessons:
- [ ] Define all technical terms
- [ ] Use simple language
- [ ] Include more examples
- [ ] Add "Why this matters" sections
- [ ] Link to prerequisite concepts

### For Advanced Lessons:
- [ ] Assume prerequisite knowledge
- [ ] Include research papers
- [ ] Add advanced experiments
- [ ] Reference latest developments
- [ ] Include optimization tips

## Time Estimates

- **Planning & Research**: 2-3 hours
- **Content Writing**: 3-5 hours
- **Sketchnote Creation**: 1-3 hours
- **Building Experiments**: 2-8 hours (varies)
- **Technical Implementation**: 1-2 hours
- **Testing & QA**: 1-2 hours
- **Total**: 10-23 hours per lesson

- [ ] Sketchnote placeholder (add image later)
- [ ] 1-2 experiment links (can use external tools)
- [ ] 5-10 curated resources
- [ ] Working navigation
- [ ] Mobile-tested
- [ ] Proofread
