📖 Introduction: Seeing Like a Cat

Have you ever watched a cat enter a new room? 🐱 Its eyes don't absorb the whole scene at once — they slide across the space, detecting edges first (is that a door frame?), then shapes (is that a box?), and finally whole objects (is that a mouse?!). It processes the world in local patches, building up understanding layer by layer.

Convolutional Neural Networks (CNNs) do exactly this with images. A tiny filter slides across an image, detecting patterns at each position — first simple edges, then textures, then full objects. Before ChatGPT could "see" your photos, a CNN was turning those pixels into numbers it could understand!

💡 Key Insight

A CNN is a neural network designed specifically for grid data like images. The word "convolution" describes the sliding-filter operation that makes it so good at detecting visual patterns — regardless of where in the image those patterns appear.

"Convolution = a tiny filter sliding across an image, lighting up wherever it finds its pattern." 🔦

📚 Core Content

The Full Name & Big Idea

A Convolutional Neural Network (CNN) is a type of deep learning model purpose-built for grid-structured data — most commonly images (a 2D grid of pixels), but also audio (1D grid of samples) and video (3D grid of frames).

The breakthrough idea: instead of connecting every pixel to every neuron (which would be astronomically large), use small shared filters that slide across the image. This exploits two key properties of images:

  • Local patterns matter: An edge looks like an edge whether it's in the top-left or bottom-right corner.
  • Hierarchy of features: Simple edges → shapes → textures → object parts → whole objects.

Why Can't a Plain Neural Net Do This?

A plain (fully-connected) network treating a 224×224 RGB image as input would need 150,528 input neurons. Connect those to just 1,000 hidden neurons → 150 million parameters in one layer alone! CNNs solve this by sharing filter weights across the whole image, slashing parameters by 99%.

📸 Instagram Face Detection

CNNs find faces in photos and suggest who to tag — even in crowd shots with hundreds of people.

🏥 Medical X-Ray Analysis

CNNs trained on millions of X-rays can detect tumours, fractures, and pneumonia faster than human radiologists.

🚗 Self-Driving Cars

Tesla's Autopilot uses CNNs to detect pedestrians, lane markings, traffic signs, and other vehicles in real time.

📱 Face ID (iPhone)

Apple's Neural Engine runs a CNN on 30,000 infrared dots projected onto your face to authenticate you in milliseconds.

The Filter (Kernel)

A filter (also called a kernel) is a tiny matrix of learnable weights — typically 3×3 or 5×5. During convolution, this filter slides across every position of the input image, and at each position computes a dot product between the filter weights and the pixel values underneath. The result is stored in the feature map.

Each filter learns to detect one specific visual pattern:

  • A vertical-edge filter: high values on left, low on right (like [ -1, 0, 1 ] repeated)
  • A horizontal-edge filter: high on top, low on bottom
  • A corner detector, a blob detector, a diagonal detector…

The network doesn't decide what to detect — training teaches it which filters are most useful!

# Visualising a vertical-edge detecting filter
import numpy as np

# A 3x3 vertical-edge filter (Sobel-ish)
filter_v = np.array([
    [-1, 0, 1],
    [-2, 0, 2],
    [-1, 0, 1]
])

# Slide it over a tiny 5x5 image (conceptual)
# At each position: sum(filter_v * patch_under_it)
# High activation → strong vertical edge found there!

🗜️ Pooling Layers

After convolution, the feature maps are still large. Max-pooling downsamples them by taking only the maximum value from each small region (e.g., 2×2 patches → single number). This:

  • Shrinks the spatial dimensions (saves compute)
  • Keeps only the strongest signal ("was this feature present anywhere in this patch?")
  • Makes the network slightly translation-invariant — a cat ear shifted a few pixels still produces a strong activation

Think of max-pooling as Whiskers' brain skimming a room: "I noticed something fluffy in the corner — I don't need to know exactly which pixel it was on." 🐾

📐 Dimension maths (for the curious):

Input: 28×28 image → 3×3 filter, stride 1, no padding → Feature map: 26×26

After 2×2 max-pool → 13×13

Formula: output_size = (input_size - filter_size) / stride + 1

Every standard CNN is built from the same four building blocks, stacked in a repeating pattern:

🔲 Convolutional Layer

Applies a bank of learnable filters to the input. Each filter produces one feature map. A typical conv layer has 32–512 filters. This is where the network learns what to look for.

⚡ ReLU Activation

Applies the function f(x) = max(0, x) — it zeros out negative values and keeps positives unchanged. This injects non-linearity, letting the network learn complex decision boundaries. Without it, stacking layers would just be matrix multiplication.

🗜️ Pooling Layer

Reduces spatial size (usually 2×2 max-pool → halves width and height). Reduces parameters, controls overfitting, and builds position tolerance. Applied after each conv+ReLU block.

🎯 Fully Connected Layer

At the end of the network, the feature maps are flattened into a 1D vector and passed through one or more fully-connected layers — just like a regular neural network. The final FC layer outputs class scores (e.g., "cat: 98%, dog: 1%, fish: 1%").

The history of CNNs is a story of going deeper — more layers, smarter tricks, better results:

📜 LeNet-5 (1998)

By: Yann LeCun at AT&T Bell Labs

Task: Handwritten digit recognition (MNIST)

Size: 5 layers, ~60,000 parameters — tiny by modern standards!

The great-great-grandparent of every modern vision model. 🐾

💥 AlexNet (2012)

By: Alex Krizhevsky et al., University of Toronto

Task: ImageNet (1.2M images, 1,000 classes)

Why famous: Won ImageNet 2012 by a huge margin using GPUs + ReLU + Dropout — launched the modern deep learning era!

📐 VGG (2014)

By: Visual Geometry Group, Oxford

Key idea: Just stack 3×3 conv layers very deep (16 or 19 layers). Simple and elegant.

Famous for: Beautifully clean architecture — great for teaching and transfer learning.

🔄 ResNet (2015)

By: Microsoft Research (He et al.)

Key idea: Skip connections (residual connections) — let gradients "skip" layers. This solved the vanishing gradient problem and enabled 152-layer networks!

Still the most-used backbone today. 🏗️

📱 MobileNet (2017)

By: Google

Key idea: Depthwise separable convolutions — 8–9× fewer operations than standard convolutions.

Designed for: Phones, embedded devices, real-time apps. Used in Google Lens!

🔍 EfficientNet (2019)

By: Google Brain

Key idea: Systematically scale depth, width, and resolution together using a compound coefficient.

Result: 8× smaller and 6× faster than the best ResNet at the same accuracy. 🚀

💻 Train Your First CNN with Keras

Let's build a CNN to classify images of cats vs. dogs using Keras (TensorFlow):

import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

# 1. Load a sample dataset (CIFAR-10 has 10 classes, 60k images)
(x_train, y_train), (x_test, y_test) = keras.datasets.cifar10.load_data()

# Normalize pixel values to [0, 1]
x_train, x_test = x_train / 255.0, x_test / 255.0

# 2. Build the CNN — Conv → ReLU → Pool, repeated twice, then FC layers
model = keras.Sequential([
    # Block 1: detect simple edges & textures
    layers.Conv2D(32, (3, 3), activation='relu', input_shape=(32, 32, 3)),
    layers.MaxPooling2D((2, 2)),

    # Block 2: detect shapes & object parts
    layers.Conv2D(64, (3, 3), activation='relu'),
    layers.MaxPooling2D((2, 2)),

    # Block 3: detect high-level features
    layers.Conv2D(128, (3, 3), activation='relu'),

    # Classifier head
    layers.Flatten(),
    layers.Dense(128, activation='relu'),
    layers.Dropout(0.5),          # prevent overfitting
    layers.Dense(10, activation='softmax')  # 10 classes
])

model.summary()
# Total params: ~470,000  (vs. 150M+ for a plain FC net!)

# 3. Compile: choose optimiser, loss function, metric
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

# 4. Train!
history = model.fit(
    x_train, y_train,
    epochs=15,
    batch_size=64,
    validation_split=0.1
)

# 5. Evaluate on test set
test_loss, test_acc = model.evaluate(x_test, y_test)
print(f"Test accuracy: {test_acc:.2%}")
# Expect ~70-75% after 15 epochs — not bad for 50 lines! 🐱

# 6. Make a prediction
import numpy as np
classes = ['airplane','automobile','bird','cat','deer',
           'dog','frog','horse','ship','truck']
sample = x_test[42:43]
pred = model.predict(sample)
print(f"Predicted: {classes[np.argmax(pred)]} ({pred.max():.1%})")
# "Predicted: cat (82.3%)" 🐾

🎯 Mini-Challenge: Transfer Learning in 10 Lines

Don't train from scratch — use a pre-trained ResNet50 and fine-tune it on your own images:

from tensorflow.keras.applications import ResNet50
from tensorflow.keras import layers, Model

# Load ResNet50 pre-trained on ImageNet (skip the top classifier)
base = ResNet50(weights='imagenet', include_top=False, input_shape=(224, 224, 3))
base.trainable = False  # Freeze base — keep ImageNet knowledge!

# Add your own classifier on top
x = layers.GlobalAveragePooling2D()(base.output)
x = layers.Dense(128, activation='relu')(x)
output = layers.Dense(2, activation='softmax')(x)  # cat vs. dog

model = Model(inputs=base.input, outputs=output)
model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])

# Expect 95%+ accuracy with just a few hundred images!  🐱
