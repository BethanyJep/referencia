📖 Introduction: How Does an AI Actually Read?

Imagine Whiskers 🐱 is just learning to read. Before she can understand a sentence, she needs to recognise each word as a distinct unit. But here's the twist — Whiskers isn't human. She can only understand numbers. So before any learning can happen, every word (and even parts of words) must be converted into a number ID. That conversion process is called tokenization.

Now imagine Cat City 🏙️🐾 — a massive neighbourhood map where every cat has an address based on its personality. Fluffy Persians live on Persian Plaza. Striped tabbies cluster on Tabby Terrace. Maine Coons hang out just a few blocks away on Maine Coon Mile. Similar cats live close to each other. That's an embedding: a numerical address (a vector) that captures meaning, so that things with similar meanings end up living near each other in the map.

Together, tokenization and embeddings are the two-step foundation that lets every modern LLM — from GPT-4 to Gemini to Claude — process human language at all.

💡 Key Insight

LLMs cannot read raw text. Every character, word, or sub-word must first become a number (tokenization), and then that number is mapped to a rich vector of meaning (embedding) before any deep learning can begin.

"Text → Tokens → Numbers → Vectors → Understanding!"

📚 Core Content

Why Tokenization Exists

Neural networks are mathematical machines — they can only operate on numbers, never on raw letters or words. Tokenization is the process of:

  1. Splitting text into manageable chunks called tokens
  2. Assigning each unique token a fixed integer ID from a vocabulary lookup table
  3. Passing those integer IDs into the model as input

Think of it as Whiskers needing a library card number for each word before she's allowed to check it out from the library! 🐱📚

The Three Main Strategies

📖 Word-Level

Each whole word is one token. Simple and intuitive, but vocabulary explodes — hundreds of thousands of entries for every language variant, typo, and conjugation.

"cats" → [5], "cat" → [4] (different tokens!)

Problem: "Whiskers-purringly" would be unknown.

🔡 Character-Level

Each individual character is one token. Tiny vocabulary (just ~100 tokens for English) but sequences become very long, and the model struggles to learn word meanings.

"cat" → ['c','a','t'] → [67, 65, 116]

Problem: Very long sequences, less efficient.

🧩 Subword (BPE / WordPiece)

The sweet spot! Splits words into common sub-pieces. Rare words break into known sub-words. Common words stay whole. Used by GPT, BERT, and most modern LLMs.

"unbelievable" → ["un", "believ", "able"]

Best of both worlds! ✅

🔢 A Real Tokenization Example

Input text: "Unbelievable!"

Tokens: ["Un", "believ", "able", "!"]

Token IDs: [1203, 4567, 891, 233]

GPT-4 uses the cl100k_base tokenizer with ~100,000 tokens in its vocabulary. That means most common English words and sub-words each have their own unique ID.

# Install: pip install tiktoken
import tiktoken

# Use the GPT-4 tokenizer
enc = tiktoken.get_encoding("cl100k_base")

text = "Unbelievable cats rule the world!"
tokens = enc.encode(text)

print("Token IDs:", tokens)
# Token IDs: [1844, 45805, 17, 19611, 279, 2088, 0]
# ... (exact IDs depend on tokenizer version)

print("Token count:", len(tokens))
# Token count: 7

# Decode back to text
decoded = enc.decode(tokens)
print("Decoded:", decoded)
# Decoded: Unbelievable cats rule the world!

Context Window = Token Budget

Every LLM has a context window — the maximum number of tokens it can process at once (input + output combined). Once you hit the limit, earlier content gets cut off or the model refuses to answer.

  • GPT-3.5: 16,384 tokens ≈ ~12,000 words
  • GPT-4: 128,000 tokens ≈ ~96,000 words (a short novel!)
  • Claude 3: 200,000 tokens ≈ a very thick textbook
  • Gemini 1.5 Pro: 1,000,000 tokens ≈ multiple books

💸 Tokens = Money

Cloud LLM APIs charge per token, not per character or word. For example, OpenAI GPT-4o:

  • Input: ~$5 per 1 million tokens
  • Output: ~$15 per 1 million tokens

A 1,000-token prompt costs about half a cent — cheap! But at scale (millions of API calls per day), it adds up fast. Efficient prompting = real money saved.

🐱 What Does 1,000 Tokens Look Like?

Roughly:

  • ~750 words of prose
  • ~3–4 paragraphs of normal English text
  • ~50 lines of Python code
  • ~2 pages of a book

Rule of thumb: 1 token ≈ 0.75 words in English. Other languages (e.g. Chinese, Arabic) often use more tokens per word!

import tiktoken

def count_tokens(text: str, model: str = "gpt-4") -> int:
    """Count tokens in a string for a specific OpenAI model."""
    enc = tiktoken.encoding_for_model(model)
    return len(enc.encode(text))

# Test with different texts
examples = [
    "Hi Whiskers!",
    "The quick brown fox jumps over the lazy dog.",
    "Explain quantum entanglement to a cat in five sentences.",
]

for text in examples:
    n = count_tokens(text)
    print(f"{n:3d} tokens | {text[:50]}")

From Token IDs to Meaning

Token IDs are just arbitrary integers — cat = 4 and kitten = 5 don't tell the model anything about their relationship. That's where embeddings come in.

An embedding is a dense vector — a list of floating-point numbers (typically 768 to 4,096 numbers long) that encodes the semantic meaning of a token. It's the model's way of saying: "Here is where this word lives in meaning-space."

🏙️ Welcome to Cat City

Picture Cat City as a 2D neighbourhood map (in reality it's 768+ dimensions, but let's keep it friendly for Whiskers):

  • 🐱 cat, kitten, feline all live on the same block
  • 🐶 dog, puppy, canine live a few streets away (related — both pets!)
  • 👑 king, queen, prince live in the Royal District
  • 🎨 art, painting, sculpture live in the Arts Quarter

Words with similar meanings get similar vector coordinates → they live near each other in the city.

✨ The Famous Analogy: king − man + woman ≈ queen

This is the most famous result in embedding research (from Word2Vec, 2013):

vector("king") - vector("man") + vector("woman")
≈ vector("queen")

Why does this work? The direction from "man" to "king" encodes the concept of royalty. Apply that same direction starting from "woman", and you land near "queen". Embeddings capture semantic relationships as geometry!

Other examples that work:

  • Paris − France + Germany ≈ Berlin
  • walked − walk + run ≈ ran
  • bigger − big + small ≈ smaller

📏 Cosine Similarity: How Close Are Two Vectors?

To measure how similar two embeddings are, we use cosine similarity:

  • 1.0 = identical meaning (same word)
  • 0.8–0.9 = very similar (cat / kitten)
  • 0.5–0.7 = related (cat / dog)
  • ~0.0 = unrelated (cat / mathematics)
  • -1.0 = opposite meaning (hot / cold)

Embeddings aren't just for words! Here are the main flavours:

📝 Word Embeddings

The classics — one vector per word. Trained on massive text corpora.

  • Word2Vec (Google, 2013)
  • GloVe (Stanford, 2014)
  • FastText (Meta, handles typos!)

Limitation: same vector for "bank" (river) and "bank" (financial) — no context!

📄 Sentence Embeddings

Encode the meaning of an entire sentence or paragraph as one vector. Much better for semantic search and document comparison.

  • SBERT (Sentence-BERT)
  • OpenAI Ada-002
  • Cohere Embed

🖼️ Image Embeddings

Images converted to vectors using CNNs or Vision Transformers. Used for reverse image search, face recognition, and image clustering.

  • CLIP (OpenAI)
  • ResNet features
  • ViT (Vision Transformer)

🌐 Multimodal Embeddings

Text and images embedded into the same vector space! You can compare a photo of a cat to the word "cat" and get a high similarity score.

  • CLIP (text + image)
  • ImageBind (Meta)
  • Gemini embeddings

🆚 Contextual vs. Static Embeddings

Early embeddings (Word2Vec, GloVe) are static — every occurrence of "bank" gets the same vector regardless of context.

Modern LLM-based embeddings are contextual — "bank" in "river bank" gets a different vector from "bank" in "savings bank". This is far more powerful!

Models like BERT, GPT, and RoBERTa produce contextual embeddings from their internal layers.

Real-World Use Cases for Embeddings

🔍 Semantic Search

Find documents by meaning, not just keywords. "How do I fix my feline's upset stomach?" finds results about cats even without the word "cat".

🎯 Recommendation Systems

Products or articles with similar embeddings get recommended together. "You liked this cat toy — here are similar ones!"

🗂️ Clustering

Group documents by topic automatically. No labels needed — similar embeddings cluster together.

🚨 Anomaly Detection

An email with a very different embedding from normal company emails might be spam or a security threat.

💻 Generate & Compare Embeddings with sentence-transformers

# pip install sentence-transformers
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

# Whiskers explores Cat City with embeddings!
sentences = [
    "Whiskers the cat loves chasing mice.",
    "My kitten enjoys hunting small rodents.",   # Similar!
    "The stock market crashed yesterday.",        # Unrelated
    "Felines are natural predators of mice.",    # Very similar!
]

embeddings = model.encode(sentences)
print(f"Embedding shape: {embeddings.shape}")
# (4, 384) — 4 sentences, 384-dimensional vectors

# Compute cosine similarity
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

query = embeddings[0]  # "Whiskers the cat loves chasing mice."
for i, (sent, emb) in enumerate(zip(sentences[1:], embeddings[1:]), 1):
    sim = cosine_sim(query, emb)
    print(f"Similarity {sim:.3f}: {sent[:50]}")

💻 Using OpenAI Embeddings API

# pip install openai
from openai import OpenAI
import numpy as np

client = OpenAI()  # Uses OPENAI_API_KEY env var

def get_embedding(text: str) -> list[float]:
    response = client.embeddings.create(
        model="text-embedding-3-small",  # 1536 dimensions
        input=text,
    )
    return response.data[0].embedding

# Embed Whiskers' cat facts
cat_embed = get_embedding("Whiskers is a fluffy orange tabby cat.")
dog_embed = get_embedding("Rex is a loyal golden retriever dog.")
food_embed = get_embedding("The lasagna was delicious at the Italian restaurant.")

# king - man + woman ≈ queen
king = get_embedding("king")
man  = get_embedding("man")
woman = get_embedding("woman")
queen_approx = np.array(king) - np.array(man) + np.array(woman)
queen_actual = get_embedding("queen")

sim = np.dot(queen_approx, queen_actual) / (
    np.linalg.norm(queen_approx) * np.linalg.norm(queen_actual)
)
print(f"king-man+woman vs queen similarity: {sim:.3f}")
# Should be ~0.85 — pretty close! 👑

💻 Mini Challenge: Build Whiskers' Semantic Search

# pip install sentence-transformers
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer("all-MiniLM-L6-v2")

# Whiskers' cat fact knowledge base
cat_facts = [
    "Cats sleep 12 to 16 hours per day on average.",
    "A cat's purring frequency promotes bone healing.",
    "Cats have 32 muscles in each ear for precise movement.",
    "Ancient Egyptians worshipped cats as sacred animals.",
    "Cats can make over 100 different vocal sounds.",
    "A group of cats is called a clowder.",
]

# Embed the knowledge base
fact_embeddings = model.encode(cat_facts, convert_to_tensor=True)

# Ask Whiskers anything!
query = "How long do cats sleep?"
query_embedding = model.encode(query, convert_to_tensor=True)

# Find the most relevant fact
scores = util.cos_sim(query_embedding, fact_embeddings)[0]
best_idx = scores.argmax().item()

print(f"Query: {query}")
print(f"Best match (score={scores[best_idx]:.2f}): {cat_facts[best_idx]}")
# Best match: "Cats sleep 12 to 16 hours per day on average." 🎯🐱
